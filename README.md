# Генератор комплектов исполнительной документации (каркас)

Локальное веб‑приложение на Flask для формирования комплектов исполнительной документации
строительных объектов. Настоящий репозиторий содержит **каркас**, реализованный по спецификации
`SPECIFICATION_FULL.md` (папка `Downloads` пользователя).

## Структура проекта

- `app.py` — точка входа Flask‑приложения, REST‑эндпоинты (пока заглушки).
- `core/` — бизнес‑логика (модули‑каркасы):
  - `utils.py` — общие утилиты (нормализация имён файлов, работа с датами).
  - `session_manager.py` — загрузка/сохранение `session.json`.
  - `template_manager.py` — работа с `.docx` и `template.json`.
  - `date_engine.py` — расчёт рабочих дней и распределение дат (частично).
  - `material_engine.py` — распределение материалов по актам (частично).
  - `renderer.py` — рендер `.docx`, конвертация в `.pdf`, merge (заглушки).
  - `registry.py` — порядок документов и реестр.
  - `errors.py` — логирование и обработка ошибок.
- `templates/` — будущие Word‑шаблоны и `template.json`.
- `sessions/` — файлы сессий (`*.json`).
- `output/` — сгенерированные документы.
- `tmp_previews/` — временные файлы предпросмотра.
- `database/` — справочники (ФИО, должности, работы и т.п.).
- `static/`, `templates_web/` — фронтенд (пока пусто).
- `logs/` — `app.log` и `error.log`.

## Установка и запуск (Windows, локально)

```bash
cd c:\AosrProject
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

После запуска приложение будет доступно по адресу `http://127.0.0.1:5000/`.

## Дальнейшая доработка

1. Реализовать логику эндпоинтов в `app.py` через вызовы модулей `core/*`.
2. Дописать `date_engine.distribute_dates` по алгоритму из спецификации.
3. Подключить `docxtpl` и `docx2pdf` в `renderer.py`.
4. Реализовать фронтенд (вкладки: Постоянные данные, Материалы, АОСР, Остальные шаблоны, Приложения, Реестр, Предпросмотр).
