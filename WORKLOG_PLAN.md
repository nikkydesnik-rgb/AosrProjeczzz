# Отчёт по выполненным работам и план дальнейшей доработки

## 1) Что уже сделано

### Шаблоны и ключи
- Реализовано извлечение ключей `{{ ... }}` из `.docx` с сохранением порядка появления.
- Добавлено разделение ключей на:
  - **постоянные** (заполняются один раз),
  - **по-актовые** (заполняются для каждого акта отдельно).
- В metadata добавлены поля для порядка ключей:
  - `constant_fields_order`,
  - `doc_fields_order`.

### Работа с реестром документов
- Добавлено добавление документов из шаблонов в реестр.
- Для шаблонов с `multiple=true` реализовано создание нескольких документов.
- Обновлена логика синхронизации порядка АОСР по полю `номер`.

### Вкладка АОСР (табличный режим)
- Вкладка АОСР переведена в **редактируемый табличный режим** (аналог Handsontable на текущем UI).
- Добавлена кнопка **"Добавить акт (строку)"**.
- Реализовано создание новой строки/акта с автозаполнением:
  - `id`,
  - `номер`,
  - `title`,
  - `order`,
  - базовые по-актовые поля.
- Добавлена двусторонняя синхронизация изменений ячеек с `session.documents`.

## 2) Что нужно сделать дальше (план)

### Ближайшие шаги (высокий приоритет)
1. Добавить удаление/дублирование строки АОСР из таблицы.
2. Добавить валидацию номера акта:
   - только положительные целые,
   - предупреждение о дубликатах.
3. Добавить визуальную подсветку конфликтов порядка в реестре.
4. Добавить явную кнопку "Синхронизировать реестр", если пользователь менял данные массово.

### Средний приоритет
1. Расширить таблицу АОСР настраиваемыми колонками из metadata `doc_fields_order`.
2. Добавить автозаполнение дат (`Ч/М/Г`, `Чнач/Мнач/Гн`) при вводе `start_date/end_date`.
3. Добавить массовые операции по выделенным строкам (копировать, вставить, очистить).

### Низкий приоритет
1. Перейти на полноценный grid-компонент (Handsontable/AG Grid) при росте объёма данных.
2. Добавить версионирование изменений по актам (история правок).
3. Подготовить экспорт/импорт только блока АОСР отдельным JSON.

## 3) Критерии готовности следующего этапа
- Пользователь может добавлять, удалять, дублировать акты без правки JSON вручную.
- Реестр стабильно совпадает с `номер` актов и валидирует конфликты.
- По-актовые поля заполняются только в таблице АОСР, постоянные — только в постоянных данных.

## 4) Риски и замечания
- Возможны конфликты `order` между АОСР и не-АОСР документами — требуется единая стратегия приоритизации.
- Требуется аккуратная миграция старых сессий, где часть полей лежит в другом формате.
- Для production-конвертации в PDF нужен платформенно-устойчивый pipeline (не только Windows+Word).
